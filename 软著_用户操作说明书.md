# 猛兽影 AI 一站式创作平台 V1.0

## 软件用户操作说明书

---

## 一、软件概述

### 1.1 软件名称
猛兽影 AI 一站式创作平台 V1.0

### 1.2 软件用途
本软件是一款基于人工智能技术的影视创作辅助平台，面向影视从业者、短视频创作者和 AI 创作爱好者。平台集成 AI 剧本智能拆分、文本生成图片、文本生成视频、数字人视频生成与推流直播等核心能力，实现从剧本编写到成片生成的全链路 AI 创作流程。

### 1.3 运行环境
- 操作系统：Windows 10/11、macOS 12+、Linux
- 浏览器：Chrome 90+、Safari 15+、Firefox 90+、Edge 90+
- 网络：需要互联网连接以调用 AI 生成服务
- 分辨率：推荐 1920×1080 及以上

### 1.4 技术架构
- 前端框架：React 18.2
- 构建工具：Vite 5.1
- 样式方案：Tailwind CSS 3.4
- 路由管理：React Router DOM 6.22

## 二、安装与部署

### 2.1 环境准备
确保已安装 Node.js 16+ 和 npm 包管理器。

### 2.2 安装步骤

1. 解压软件源代码包至目标目录
2. 打开终端，进入项目根目录
3. 执行依赖安装：
   ```
   npm install
   ```
4. 启动开发服务器：
   ```
   npm run dev
   ```
5. 浏览器访问 http://localhost:5173 即可使用

### 2.3 生产部署
```
npm run build
```
构建产物位于 `dist/` 目录，可部署至任意静态文件服务器（Nginx、Apache、CDN 等）。

## 五、大模型·剧本

### 5.1 功能说明
本模块用于将完整剧本智能拆分为结构化的剧集和分镜，并自动提取角色信息。支持上传 TXT、MD、DOCX 格式剧本文件。

### 5.2 操作流程

#### 5.2.1 上传剧本
1. 点击「上传剧本」按钮
2. 选择剧本文件（支持 .txt、.md、.docx 格式）
3. 系统自动读取并显示剧本内容

#### 5.2.2 配置参数
- **工作项目**：填写当前项目名称，用于标识
- **固定关键词**：设定每次请求中固定携带的关键词（如风格、年代等）
- **上下文记忆**：填写需要 AI 记住的上下文信息

#### 5.2.3 AI 剧本拆分
1. 剧本上传完成后，点击「AI 拆分」按钮
2. 系统将剧本内容连同固定关键词、上下文记忆一起发送给大模型
3. AI 返回结构化的剧集和分镜数据
4. 每个分镜包含 11 个影视要素：
   - 景别（远景/全景/中景/近景/特写）
   - 构图（三分法/对称/前景遮挡等）
   - 运镜（推/拉/摇/移/跟/升/降/固定）
   - 演员调度（走位、互动、入画出画）
   - 色调
   - 光影
   - 机位
   - 特效
   - 备注
   - 描述（prompt）
   - 台词/旁白

#### 5.2.4 AI 角色提取
1. 点击「提取角色」按钮
2. AI 自动从剧本中识别角色信息（姓名、描述等）
3. 提取结果展示在角色列表区

#### 5.2.5 批量导出到文生图
1. 拆分完成后，点击「批量生成图片」按钮
2. 系统将所有分镜的 prompt 列表传递到文生图页面
3. 在文生图页面可逐镜生成或批量生成

### 5.3 数据说明
- 固定关键词、上下文记忆、工作项目保存在浏览器本地
- 分镜数据保存在会话存储中，关闭页面后清除

## 六、文生图

### 6.1 功能说明
本模块采用 Cinema Studio 设计风格，支持通过文本描述生成 AI 图片。用户可配置比例、画质、摄影参数（机型、镜头、焦距、光孔），所有视觉相关设置会自动融入 AI 提示词。

### 6.2 操作流程

#### 6.2.1 输入描述
在文案输入框中输入想要生成的画面描述，例如：
```
一位穿旗袍的女子走在雨后的青石板路上，黄昏光线，电影质感
```

#### 6.2.2 配置比例
点击底部工具栏的比例按钮，可选择：
- 21:9（电影宽银幕）
- 16:9（标准宽屏）
- 9:16（竖屏/手机）
- 1:1（方形）
- 4:3（传统比例）
- 3:4（竖版 4:3）

#### 6.2.3 配置画质
点击画质按钮，可选择：
- 标准 1K
- 高清 2K（默认）
- 超清 4K

#### 6.2.4 摄影参数
点击「📷 摄影」按钮打开摄影参数面板：

1. **推荐预设**：点击可一键应用
   - 电影变形宽银幕（Super 35 + Anamorphic + 35mm + f/4）
   - 暖调定焦（Super 35 + Warm Cinema Prime + 50mm + f/2.8）
   - 人像特写（Full Frame + Standard Prime + 85mm + f/2）

2. **自定义参数**：
   - 机型：Studio Digital S35 / 全画幅 / APS-C / Super 35 / 大画幅
   - 镜头：Warm Cinema Prime / Anamorphic / Standard Prime 等
   - 焦距：自由输入（单位 mm）
   - 光孔：f/1.4 ~ f/22

#### 6.2.5 文字输出标记
点击「T」按钮，在光标位置插入 `""`，引号内填写需要在画面中显示的文字。

#### 6.2.6 参考图上传
点击「+ 参考」按钮上传参考图片，AI 将参考已上传图片的风格进行生成。

#### 6.2.7 生成图片
点击「生成」按钮，系统将：
1. 将用户描述 + 比例 + 摄影参数拼合为完整提示词
2. 连同画质等结构化参数发送给 AI 服务
3. 在预览区显示生成结果

### 6.3 提示词拼接示例
```
一位穿旗袍的女子走在雨后的青石板路上
[设置] 比例 21:9 | 摄影 Full Frame, Warm Cinema Prime, 50mm, f/2.8
```

## 七、视频生成

### 7.1 功能说明
本模块支持通过文本描述生成 AI 视频，提供四种参考模式、可调比例/时长、画质/格式/编码等输出设置。

### 7.2 四种生成模式

#### 7.2.1 全能参考
- 上传 1~5 个参考图片或视频
- 在描述中用 `@图片1`、`@视频1` 关联参考素材
- 适合需要指定风格、角色或场景的创作

#### 7.2.2 首尾帧
- 上传首帧图片和尾帧图片
- AI 生成从首帧到尾帧的过渡视频
- 适合需要精确控制起止画面的场景

#### 7.2.3 智能多帧
- 上传 1~8 个关键帧
- AI 在关键帧之间智能补间生成视频
- 适合需要多个关键节点的长镜头

#### 7.2.4 主体参考
- 上传主体参考图
- AI 保持主体一致性生成视频

### 7.3 输出设置

| 设置项 | 选项 | 传递方式 |
|--------|------|----------|
| 比例 | 21:9 / 16:9 / 9:16 / 1:1 / 4:3 / 3:4 | 写入提示词 + payload |
| 时长 | 5s ~ 15s | 写入提示词 + payload |
| 画质 | 高清 / 2K / 4K | payload 结构化字段 |
| 格式 | MOV / MP4 | payload 结构化字段 |
| 编码 | H.264 / H.265 | payload 结构化字段 |

### 7.4 操作流程
1. 选择生成模式（全能参考/首尾帧/智能多帧/主体参考）
2. 输入文字描述
3. 按模式要求上传参考素材
4. 调整比例、时长、画质、格式、编码
5. 点击「生成」按钮
6. 生成完成后，结果卡片展示参考缩略图、视频预览、时长、详细参数
7. 可下载或重新生成

## 八、数字人

### 8.1 功能说明
本模块提供两种模式：单图视频生成和推流直播。

### 8.2 单图视频生成
1. 上传角色形象图片
2. 选择或上传音色文件
3. 输入说话文案
4. 输入动作描述
5. 配置输出设置（画质/格式/编码）
6. 点击「生成视频」

### 8.3 推流直播
1. 切换至「推流直播」模式
2. 填写推流地址（RTMP URL）
3. 填写推流密钥（Stream Key）
4. 配置角色形象和音色
5. 点击「开始推流」
6. 推流中离开页面会弹出确认提示，防止误关

### 8.4 输出设置
- 画质：高清 / 2K / 4K
- 格式：MOV / MP4
- 编码格式：H.264 / H.265

## 九、API 对接

### 9.1 功能说明
本模块用于配置各 AI 服务的 API 连接信息。剧作大模型与视频/图/数字人使用不同的后端服务，可分别配置。

### 9.2 通用配置
- **Base URL**：视频/图/数字人共用的服务根地址
- **API Key**：可选的 Bearer Token 认证密钥

### 9.3 服务端点配置

| 服务 | 默认端点 | 说明 |
|------|---------|------|
| 剧作大模型 | /v1/script/breakdown | 可配独立 Base URL |
| 角色提取 | /v1/script/characters | 可选，留空用本地规则 |
| 图片生成 | /v1/image/generate | 使用通用 Base URL |
| 视频生成 | /v1/video/generate | 使用通用 Base URL |
| 数字人 | /v1/digital-human | 使用通用 Base URL |

### 9.4 连通性检测

#### 单个检测
每个服务右侧有「⚡ 检测连通」按钮，点击后系统会：
1. 依次尝试 HEAD / GET / POST 方法探测端点
2. 8 秒超时保护
3. 结果通过指示灯显示：
   - 🟢 绿色亮灯（带光晕脉冲）= 已连通
   - 🔴 红色亮灯（带光晕脉冲）= 连接失败
   - 🔵 蓝色旋转 = 检测中
   - ⚫ 灰色 = 未检测

#### 一键全部检测
点击「🔌 一键检测全部服务」按钮，所有已启用服务并发检测，互不阻塞。

### 9.5 配置存储
所有配置保存在浏览器本地存储（localStorage），键名为 `jiangying_api_config`。

## 十、界面说明

- 固定在页面顶部，采用毛玻璃效果

## 十一、数据安全说明

1. 所有用户配置数据存储在浏览器本地（localStorage），不会上传至第三方服务器
2. API Key 在页面中以密码形式显示，不会明文暴露
3. 登录状态仅保存在客户端，关闭浏览器并清除缓存后需重新登录
4. AI 生成请求直接发送至用户自行配置的 API 服务地址，平台不中转数据

## 十二、常见问题与解决方案

| 问题 | 解决方案 |
|------|---------|
| 页面空白或加载失败 | 检查 Node.js 版本 ≥ 16，重新执行 npm install |
| Logo 显示异常 | 强制刷新浏览器（Ctrl+Shift+R / Cmd+Shift+R） |
| API 检测失败 | 检查 Base URL 是否正确、后端服务是否启动、网络是否通畅 |
| 剧本上传后无内容 | 检查文件格式是否为 .txt / .md / .docx |
| 生成结果为空 | 检查 API 配置页服务端点是否正确配置并已连通 |
| 推流中误关页面 | 系统会弹出确认提示，请确认后再关闭 |
