# 猛兽影 AI 一站式创作平台 V1.0

## 软件设计说明书

---

## 一、引言

### 1.1 编写目的
本文档描述「猛兽影 AI 一站式创作平台 V1.0」的软件架构设计、模块划分、数据流设计、接口规范等，作为软件著作权登记的技术说明文档。

### 1.2 软件概述
猛兽影 AI 是一款基于 Web 技术的 AI 影视创作辅助平台，采用前后端分离架构。前端负责用户交互、提示词拼装与结果展示；后端通过可配置的 API 接口提供 AI 生成能力。

### 1.3 开发环境

| 项目 | 说明 |
|------|------|
| 开发语言 | JavaScript (ES2022+)、JSX、CSS |
| 前端框架 | React 18.2 |
| 构建工具 | Vite 5.1 |
| CSS 框架 | Tailwind CSS 3.4 |
| 路由框架 | React Router DOM 6.22 |
| 文档解析 | mammoth 1.6（按需动态加载） |
| 包管理器 | npm |
| 版本控制 | Git |

│  ├──────────────────────────────────────────────────┤│

### 2.2 技术选型理由

| 技术 | 选型理由 |
|------|---------|
| React 18 | 组件化开发、虚拟 DOM 高性能、生态丰富 |
| Vite 5 | 极速冷启动、ESM 原生支持、Tree-shaking |

## 三、模块设计

### 3.1 模块总览

```
src/
├── main.jsx               # 应用入口模块
├── App.jsx                # 路由调度模块
├── context/
│   └── AuthContext.jsx    # 认证状态管理模块
├── components/
│   ├── auth/              # 认证守卫模块
│   ├── layout/            # 布局模块（导航栏、主布局）
│   └── ui/                # 通用 UI 组件模块
└── pages/                 # 业务页面模块（9个）
```

### 3.2 入口模块（main.jsx）

**职责**：初始化 React 应用，挂载根组件，配置全局 Provider。

**依赖关系**：
- BrowserRouter（路由容器）
- AuthProvider（认证上下文）
- App（路由调度）
- index.css（全局样式）

### 3.3 路由调度模块（App.jsx）

**职责**：配置全局路由表，实现页面级懒加载。

**设计要点**：
- 使用 `React.lazy()` + `Suspense` 实现 9 个页面的按需加载
- 登录保护通过 `AuthGuard` 组件实现嵌套路由守卫
- `MainLayout` 组件提供统一的导航栏和动态背景

**路由层级**：
```
/                          → LandingPage（公开）
/dev                       → DeveloperLandingPage（公开）
/login                     → LoginPage（公开）
<AuthGuard>                → 需登录
  <MainLayout>             → 统一布局
    /ai-tool/home          → HomePage
    /ai-tool/script        → ScriptLLMPage
    /ai-tool/image/generate → ImageGeneratePage
    /ai-tool/video         → VideoGeneratePage
    /ai-tool/digital-human → DigitalHumanPage
    /ai-tool/api           → ApiPage
```

### 3.4 认证模块（AuthContext.jsx）

**职责**：全局管理用户登录/登出状态。

**接口**：
- `login(username, password)` → boolean：登录验证
- `logout()` → void：清除登录状态
- `user` → string | null：当前用户名
- `isLoggedIn` → boolean：是否已登录

**持久化**：通过 localStorage 键 `jiangying_auth_user` 存储用户名，刷新页面保持登录。

### 3.5 布局模块

#### Header 组件
- 品牌 Logo（高度随 CSS 变量 `--header-h` 自适应）
- 导航菜单（6 项，当前页高亮）
- 用户信息与操作按钮

#### MainLayout 组件
- 动态背景层（DynamicBackground）
- 导航栏（Header）
- 内容区（Outlet）

#### DynamicBackground 组件
- 5 个绿色光球（CSS keyframes 漂移 + 呼吸）
- 电路科技纹理层（SVG 平铺，opacity 0.6）
- 光照层（与光球同步，mix-blend-mode: screen 照亮电路）

### 3.6 通用 UI 组件模块

| 组件 | 功能 |
|------|------|
| Button | 按钮（primary/ghost/outline，支持 size） |
| Input | 输入框（text/password/textarea，forwardRef） |
| Card | 卡片容器（毛玻璃效果） |
| Badge | 标签/徽标（default/primary/success） |
| Modal | 模态弹窗 |
| Tab | 选项卡切换 |

### 3.7 业务页面模块

#### ScriptLLMPage（大模型·剧本）
- 文件读取：txt/md 用 FileReader，docx 用 mammoth（动态 import）
- API 调用：剧本拆分、角色提取
- 数据结构：episodes[] → shots[]（11 要素）
- 批量导出：sessionStorage 存储 prompt 列表

#### ImageGeneratePage（文生图）
- 提示词拼装：用户文案 + 比例 + 摄影参数
- 摄影参数：机型/镜头/焦距/光孔，推荐预设
- Portal 渲染：画质下拉、摄影面板挂载到 document.body
- API 调用：发送完整 prompt + 结构化参数

#### VideoGeneratePage（视频生成）
- 四种模式：全能参考/首尾帧/智能多帧/主体参考
- 富文本编辑：contentEditable + @关联参考
- 提示词拼装：用户文案 + 比例 + 时长 + 模式
- 结果管理：生成结果卡片列表，支持下载/删除

#### DigitalHumanPage（数字人）
- 双模式：单图视频 / 推流直播
- 离开保护：推流中 beforeunload 提示
- 输出参数：画质/格式/编码通过 payload 传递

#### ApiPage（API 对接）
- 分层配置：通用 Base URL + 各服务独立端点
- 连通检测：probeEndpoint 函数（HEAD→GET→POST，8s 超时）
- 状态指示：StatusLight 组件（绿/红/蓝/灰四态灯）
- 并发检测：Promise.allSettled 一键全部检测

## 四、数据流设计

### 4.1 提示词数据流

```
用户输入描述 ──┐
               ├──→ getFullPrompt() ──→ AI 提示词
选定的设置 ────┘        │
                        ├──→ prompt 字段（AI 可理解的：比例、摄影、时长）
                        └──→ payload 结构化字段（后端处理的：画质、格式、编码）
```

### 4.2 提示词拼接规则

| 页面 | 写入 prompt 的设置 | 仅 payload 传递的设置 |
|------|-------------------|---------------------|
| 文生图 | 比例、摄影参数 | 画质 |
| 视频生成 | 比例、时长、模式 | 画质、格式、编码 |
| 数字人 | （无） | 画质、格式、编码 |

**设计原则**：AI 能通过提示词理解的视觉设置写入 prompt；后处理参数（编码、封装、分辨率）仅通过结构化字段传递给后端。

### 4.3 API 配置数据流

```
API 配置页 ──保存──→ localStorage['jiangying_api_config']
                           │
        ┌──────────────────┼──────────────────┐
        ↓                  ↓                  ↓
   剧本拆分页          文生图/视频页         数字人页
   (读取 scriptLLM     (读取 baseUrl        (读取 baseUrl
    独立 baseUrl)       + 各 endpoint)       + endpoint)
```

### 4.4 认证数据流

```
登录页 ──login()──→ AuthContext ──→ localStorage['jiangying_auth_user']
                       │
                  AuthGuard 检查
                   ├── 已登录 → 渲染子路由
                   └── 未登录 → 重定向 /login
```

## 五、接口设计

### 5.1 前端发出的 API 请求

所有请求均为 `POST`，`Content-Type: application/json`。

#### 图片生成接口
```
POST {baseUrl}{image.endpoint}
Headers: Authorization: Bearer {apiKey}  （如配置）
Body: {
  prompt: string,    // 完整提示词（含[设置]后缀）
  ratio: string,     // 比例如 "21:9"
  quality: string,   // 画质如 "2k"
  camera?: string    // 摄影参数字符串
}
```

#### 视频生成接口
```
POST {baseUrl}{video.endpoint}
Body: {
  prompt: string,       // 完整提示词
  duration: number,     // 时长（秒）
  aspectRatio: string,  // 比例
  resolution: string,   // 画质
  format: string,       // 封装格式 mov|mp4
  codec: string,        // 编码 h264|h265
  frameMode: string     // 模式
}
```

#### 数字人接口
```
POST {baseUrl}{digitalHuman.endpoint}
Body: {
  speakContent: string,  // 说话内容
  actionDesc: string,    // 动作描述
  resolution: string,    // 画质
  format: string,        // 格式
  codec: string          // 编码
}
```

#### 剧本拆分接口
```
POST {scriptLLM.baseUrl}{scriptLLM.endpoint}
Body: {
  project: string,
  fixedKeywords: string,
  contextMemory: string,
  script: string
}
```

#### 角色提取接口
```
POST {scriptLLM.baseUrl}{scriptLLM.charactersEndpoint}
Body: { script: string }
```

### 5.2 连通性检测接口

```javascript
probeEndpoint(url, apiKey)
  → 依次尝试 HEAD / GET / POST
  → 8 秒超时
  → 返回 { ok: boolean, message: string }
  → 401/403 视为连通（服务存在但需鉴权）
```

## 六、性能设计

### 6.1 代码分割策略

| 策略 | 实现方式 | 效果 |
|------|---------|------|
| 路由级懒加载 | React.lazy + Suspense | 首页仅加载当前页 JS |
| 三方库分包 | Vite manualChunks | React/Router 独立缓存 |
| 大库按需加载 | mammoth 动态 import() | 478KB 仅上传 docx 时下载 |

### 6.2 渲染性能

| 优化项 | 说明 |
|-------|------|
| CSS 动画 | 背景动效全用 CSS keyframes，不占用 JS 主线程 |
| will-change | 光球元素标记 will-change: transform, opacity |
| pointer-events: none | 背景层不拦截点击事件 |
| useCallback/useMemo | 关键函数和计算缓存，避免不必要的重渲染 |

### 6.3 构建产物

| 类型 | 文件数 | 总大小 |
|------|-------|-------|
| 入口 + 框架 | 3 | ~172 KB |
| 页面模块 | 9 | ~97 KB |
| 按需库 | 1 | ~494 KB（仅 docx 时加载） |
| CSS | 1 | ~37 KB |

## 七、安全设计

| 安全措施 | 说明 |
|---------|------|
| 密码隐藏 | Input 组件支持 type="password"，密码不明文显示 |
| 本地存储 | 配置数据仅存浏览器端，不上传第三方 |
| API Key 保护 | 配置页以密码形式显示，请求中通过 Authorization Header 传递 |
| 路由守卫 | AuthGuard 阻止未登录用户访问功能页面 |
| 离开保护 | 数字人推流中关闭页面前弹出确认提示 |
