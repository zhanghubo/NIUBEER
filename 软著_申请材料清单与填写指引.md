### 2.3 软件用途和技术特点（模板）

```
本软件是一款基于人工智能技术的影视创作辅助平台。采用 React 前端框架和
Vite 构建工具开发，支持 AI 剧本智能拆分（按分集/人物/场景11要素结构化）、
文本生成图片（Cinema Studio 摄影参数融入提示词）、文本生成视频（四种参考
模式，支持高清/2K/4K画质，MOV/MP4格式，H.264/H.265编码）、数字人视频
生成与推流直播等核心功能。软件采用前后端分离架构，通过可配置的 API 端点
灵活对接各类大模型服务，内置服务连通性检测机制。采用 React.lazy 路由级
懒加载和第三方库分包策略优化加载性能。
```

### 2.4 主要功能（模板）

```
1. AI 剧本拆分：上传 TXT/MD/DOCX 剧本，AI 自动拆分为剧集和分镜（含景别、
   构图、运镜等11个影视要素），支持 AI 角色信息提取。
2. AI 文生图：输入文本描述，配置画面比例和摄影参数（机型、镜头、焦距、光孔），
   AI 自动生成高质量图片，支持推荐预设和参考图上传。
3. AI 视频生成：支持全能参考、首尾帧、智能多帧、主体参考四种模式，可配置
   比例、时长、画质、格式和编码，生成高质量 AI 视频。
4. 数字人：支持单图视频生成（角色形象 + 说话内容 + 动作描述 → 视频）和
   推流直播（RTMP 推流 → 实时数字人直播）两种模式。
5. API 对接：可配置通用 Base URL 和各服务独立端点，内置连通性检测功能
   （指示灯 + 一键全部检测），支持剧作大模型独立地址配置。
6. 用户认证：登录/登出管理，路由级权限守卫。
7. 性能优化：路由级懒加载、第三方库分包、大型库按需动态加载。
```

## 三、源程序文档要求

### 3.1 格式要求
- 前 30 页 + 后 30 页（共 60 页）
- 每页不少于 50 行
- 如总页数不足 60 页，提交全部源代码
- 不含空白页

### 3.2 注意事项
- 源代码应为**可编译/可运行的实际代码**
- 不能包含第三方库源码（node_modules 下的代码不能提交）
- 只包含自己编写的代码
- 本项目已生成的 `软著_源程序文档.txt` 符合上述要求

## 四、文档材料要求

### 4.1 可选文档类型（二选一）
1. **用户操作说明书**（推荐）：描述软件的功能、使用方法、操作界面等
2. **软件设计说明书**：描述软件架构、模块设计、数据流等

### 4.2 格式要求
- 不少于 15 页（建议 20 页以上）
- 需包含软件界面截图或界面说明
- 文档页眉标注软件名称
- 每页有页码

### 4.3 本项目已准备
- `软著_用户操作说明书.md`：约 20+ 页，涵盖所有功能的详细操作说明
- `软著_设计说明书.md`：约 15+ 页，涵盖架构、模块、数据流、接口、性能设计

**提交时建议二选一**，推荐提交用户操作说明书（更直观、更容易审核通过）。

## 五、提交流程

### 步骤一：在线填写申请表
1. 登录 https://www.ccopyright.com.cn
2. 选择「我要登记」→「软件著作权登记申请」
3. 按上方指引填写各字段
4. 保存并打印申请表
5. 著作权人签名/盖章

### 步骤二：准备纸质材料
1. 打印源程序文档（`软著_源程序文档.txt`，A4 纸）
2. 打印文档材料（用户操作说明书或设计说明书，A4 纸）
3. 复印身份证明（个人身份证正反面 / 企业营业执照副本）
4. 如有代理人，准备委托书

### 步骤三：排版建议
- **源程序文档**：
  - 页眉：猛兽影AI一站式创作平台 V1.0 — 源程序
  - 字体：等宽字体（如 Courier New / 宋体），9~10pt
  - 每页 50 行
  - 页脚标注页码

- **文档材料**：
  - 页眉：猛兽影AI一站式创作平台 V1.0 — 用户操作说明书
  - 字体：宋体/黑体，正文小四号
  - 标题分级明确
  - 插入软件界面截图（建议运行 npm run dev 后截图）

### 步骤四：提交
- **线上提交**：上传电子版至版权保护中心系统
- **线下提交**：邮寄或现场提交纸质材料至中国版权保护中心

### 步骤五：等待审核
- 审核周期：一般 30~60 个工作日（加急另计）
- 审核通过后领取《计算机软件著作权登记证书》
